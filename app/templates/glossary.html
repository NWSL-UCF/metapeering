Glossary:
  • Possible Peering Contracts (PPCs) lists all possible peering points where two ISPs can peer regardless of the in/out-bound traffic ratio.
  • Acceptable Peering Contracts (APCs) sorts all PPCs according to the requester sorting preference and suggests the top three.
Acceptable Peering Contracts sorting criteria:
  • Difference Minimizes the absolute difference between in/out-bound traffic for requester.
  • Own Maximizes the requester ISPâ€™s own traffic amount that it can offload, regardless of how much traffic it receives from the candidate.
  • Ratio Choose peers with lower in/out-bound traffic ratio.
Peering query input format:
  • ASN1 needs to be an integer, cannot be left empty.
  • ASN2 needs to be an integer, if left empty, will return a list of all ISPs peering is recommended with. If given a valid value, will show if peering is recommended.
  • Threshold needs to be a decimal between 0.0 and 1.0, if left empty, will use the default value (0.5).
